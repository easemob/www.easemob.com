loadMoreMessagesFromDB

  • loadMoreMessagesFromDB(startMsgId?, pageSize?, direction?): Promise<ChatMessage[]>

  • 从指定消息 ID 开始分页加载数据库中的消息。

    Parameters

    • startMsgId: string = ''

      查询的起始消息 ID。该参数设置后，SDK 从指定的消息 ID 开始按消息检索方向加载。默认值为空字符串。 如果传入消息的 ID 为空，SDK 忽略该参数，按搜索方向查询消息： - 若 direction 为 UP，SDK 从最新消息开始，按消息时间戳的倒序获取； - 若 direction 为 DOWN，SDK 从最早消息开始，按消息时间戳的正序获取。

    • pageSize: number = 20

      每页要加载的消息数。取值范围为 [1,400]。默认值为 20。

    • direction: SearchDirection = SearchDirection.UP

      消息搜索方向。 - （默认）UP：按照消息中的时间戳的逆序查询； - DOWN：按照消息中的时间戳的正序查询。

    Returns Promise<ChatMessage[]>

    • 消息列表（不包含查询起始 ID 的消息）。

searchMessagesByKeywords

  • searchMessagesByKeywords(keywords, timestamp?, maxCount?, from?, direction?, searchScope?): Promise<ChatMessage[]>

  • 从本地数据库获取会话中的指定用户发送的包含特定关键词的消息。

    Parameters

    • keywords: string

      查询的关键字。

    • timestamp: number = -1

      查询的起始消息 Unix 时间戳，单位为毫秒。该参数设置后，SDK 从指定时间戳的消息开始，按消息搜索方向获取。 如果该参数设置为负数，SDK 从当前时间开始搜索。默认值为 -1。

    • maxCount: number = 20

      每次获取的最大消息数。取值范围为 [1,400]。默认值为 20。

    • from: string | string[] = ''

      单聊或群聊中的消息发送方的用户 ID 或者用户 ID 数组。若设置为空字符串，SDK 将在整个会话中搜索消息。默认值为空字符串。

    • direction: SearchDirection = SearchDirection.UP

      消息搜索方向。 - （默认）UP：按照消息中的时间戳的逆序查询； - DOWN：按照消息中的时间戳的正序查询。

    • searchScope: MessageSearchScope = MessageSearchScope.ALL

      消息搜索范围。默认按消息内容和扩展属性搜索。默认值为MessageSearchScope.ALL。

    Returns Promise<ChatMessage[]>

    • 消息列表（不包含查询起始时间戳对应的消息）。
