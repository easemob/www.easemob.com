Class ChatManager

聊天管理类，该类负责管理会话（加载，删除等）、发送消息、下载消息附件等。

发送文本消息示例如下：

let textMessage = ChatMessage.createTextSendMessage(content, toChatUserId);
ChatClient.getInstance().chatManager()?.sendMessage(textMessage);

ackMessageRead

  • ackMessageRead(to, messageId): void

  • 发送消息的已读回执。

    该方法仅适用于单聊会话，仅在 ChatOptions#setRequireReadAck 为 true 时生效。

      已读回执的接收人的用户 ID。

    Returns void

downloadAndParseCombineMessage

  • downloadAndParseCombineMessage(message): Promise<ChatMessage[]>

  • 下载并解析合并消息的附件。

    对于一条合并消息，首次调用该方法会下载和解析合并消息附件，然后返回消息列表，而后续调用会存在以下情况：

fetchConversationsFromServer

  • fetchConversationsFromServer(limit, cursor): Promise<CursorResult<Conversation>>

  • 分页从服务器获取获取会话列表。

    SDK 按照会话活跃时间（会话的最后一条消息的时间戳）倒序返回会话列表。

    若会话中没有消息，则 SDK 按照会话创建时间的倒序返回会话列表。

    Parameters

    • limit: number

      每页返回的会话数量。取值范围为 [1,50]。

    • cursor: string

      开始获取数据的游标位置。若获取数据时传空字符串（""），SDK 从最新活跃的会话开始获取。

    Returns Promise<CursorResult<Conversation>>

    • 返回分页获取的 Promise 结果。

modifyMessage

  • modifyMessage(messageId, modifiedBody?, ext?): Promise<ChatMessage>

  • 修改消息内容。

    调用该方法修改消息内容后，本地和服务端的消息均会修改。

    • 文本/自定义消息：支持修改消息内容（body）和扩展 ext。
    • 文件/视频/语音/图片/位置/合并转发消息：只支持修改消息扩展 ext。
    • 命令消息：不支持修改。

    该方法会同时更新服务器和本地的消息，消息 ID 不会更新。

    多设备和该消息的接收方可通过 ChatMessageListener#onMessageContentChanged 回调监听消息是否有变更。

    Parameters

    • messageId: string

      消息 ID。

    • OptionalmodifiedBody: TextMessageBody | CustomMessageBody

      内容修改后的消息体（可选）。

    • Optionalext: any

      扩展信息（可选）。

    Returns Promise<ChatMessage>

    • 消息修改是否成功的 Promise 结果。
